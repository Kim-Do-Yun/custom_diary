================================================================================
                        Studio9910 프로젝트 요구사항 분석서
================================================================================

프로젝트명: Studio9910 - AI 기반 다이어리 웹툰 생성 앱
작성일: 2024년 12월
버전: 1.0 (회원가입 플로우 완성 단계)
플랫폼: Android (Kotlin)

================================================================================
1. 프로젝트 개요
================================================================================

1.1 프로젝트 목적
- 사용자의 일상 다이어리를 생성형 AI를 활용하여 개인 맞춤형 웹툰으로 변환해주는 Android 앱
- 사용자 개인의 이야기와 감정을 시각적 스토리텔링으로 재탄생시키는 창작 플랫폼 제공
- AI 기술과 웹툰 문화를 결합한 혁신적인 일기 작성 경험 구현

1.2 전체 프로젝트 범위 (최종 목표)
- 일기 작성 및 관리 시스템
- AI 기반 웹툰 자동 생성 엔진
- 사용자 맞춤형 그림체 및 스토리 스타일 적용
- 생성된 웹툰 저장, 편집, 공유 기능
- 개인 웹툰 아카이브 및 갤러리

1.3 현재 완성된 범위 (Phase 1)
- **회원가입 플로우**: Toss 스타일 "one page one action" 디자인 적용
- **사용자 프로필 수집**: AI 웹툰 생성을 위한 기초 데이터 수집
- **선호도 분석**: 장르 및 그림체 선택을 통한 개인화 기반 구축

1.4 대상 사용자
- 일기 작성을 좋아하지만 새로운 경험을 원하는 사용자
- 웹툰 및 만화 콘텐츠를 즐기는 모든 연령대
- 개인 스토리를 시각적으로 표현하고 싶어하는 창작 지향 사용자
- AI 기술을 활용한 개인화 서비스에 관심이 있는 사용자

2. 기능 요구사항 (Functional Requirements)
================================================================================

2.1 회원가입 플로우 (현재 완성 단계)

2.1.1 회원가입 단계별 순서
1. 약관 동의 (SignupTermsActivity) - 서비스 이용 및 개인정보 처리 동의
2. 이메일 입력 (SignupEmailActivity) - 계정 생성을 위한 기본 인증 정보
3. 비밀번호 설정 (SignupPasswordActivity) - 보안 강화를 위한 비밀번호 생성
4. 비밀번호 확인 (SignupPasswordConfirmActivity) - 비밀번호 정확성 검증
5. 개인정보 입력 (SignupProfileActivity) - AI 개인화를 위한 기본 프로필
6. 장르 선택 (SignupGenreActivity) - **AI 웹툰 생성 시 스토리 스타일 결정**
7. 그림체 선택 (SignupArtStyleActivity) - **AI 웹툰 생성 시 비주얼 스타일 결정**

2.1.2 플로우 특징 및 AI 연동 목적
- 각 페이지는 하나의 주요 액션만 수행 (One Page One Action) - 사용자 피로도 최소화
- 조건 충족 시에만 다음 단계로 진행 가능 - 데이터 품질 보장
- **수집된 선호도 정보는 AI 웹툰 생성 알고리즘의 기초 데이터로 활용**
- 부드러운 화면 전환으로 몰입감 있는 온보딩 경험 제공

2.2 약관 동의 기능 (SignupTermsActivity)

2.2.1 기본 요구사항
- AI 서비스 이용을 위한 필수 약관 동의 수집
- 개인 다이어리 데이터 처리를 위한 개인정보 이용 동의
- 생성형 AI 활용을 위한 서비스 특화 약관 포함

2.2.2 약관 구성
- **필수 약관**: 
  * 서비스 이용약관 (AI 웹툰 생성 서비스 포함)
  * 개인정보 수집 및 이용 동의 (다이어리 텍스트, 개인 선호도 포함)
  * 위치기반 서비스 이용약관
- **선택 약관**:
  * 마케팅 정보 수신 동의
  * 이벤트 및 프로모션 알림 동의
  * 제3자 정보 제공 동의

2.2.3 상세 기능
- 계층적 체크박스 시스템으로 사용자 편의성 제공
- 전체 동의 시 모든 하위 항목 일괄 처리
- 필수 약관 모두 동의 시에만 서비스 이용 가능

2.3 이메일 입력 기능 (SignupEmailActivity)

2.3.1 기본 요구사항
- AI 웹툰 서비스 계정 생성을 위한 기본 인증 정보 수집
- 생성된 웹툰 및 서비스 알림 발송을 위한 연락처 확보
- 실시간 이메일 형식 검증 및 중복 확인

2.3.2 검증 로직
- 이메일 형식 검증 (Android Patterns.EMAIL_ADDRESS 사용)
- 서버 중복 확인 (기존 사용자와 구분을 위한 유니크 체크)
- 디바운싱 적용 (500ms 지연) - 불필요한 서버 호출 최소화
- 서버 응답 시뮬레이션 (1초) - 실제 서비스 환경 대비

2.3.3 UI 상태 관리
- 실시간 피드백으로 사용자 경험 향상
- 로딩, 성공, 실패 상태별 명확한 시각적 구분
- 에러 메시지를 통한 사용자 가이드 제공

2.4 비밀번호 설정 기능 (SignupPasswordActivity)

2.4.1 보안 요구사항
- AI 서비스 및 개인 다이어리 데이터 보호를 위한 강력한 비밀번호 정책
- 실시간 조건 검증으로 사용자 편의성과 보안성 동시 확보

2.4.2 비밀번호 조건 (보안 강화)
1. 길이: 8~16자 이내 (개인 데이터 보호를 위한 충분한 길이)
2. 구성: 특수문자와 소문자 포함 (복잡도 확보)
3. 제한: 공백 미포함 (입력 오류 방지)

2.4.3 UI 피드백
- 각 조건별 실시간 시각적 피드백 (체크/X 아이콘)
- 조건 충족 시 즉시 확인 가능한 색상 변화
- 모든 조건 충족 시에만 다음 단계 진행 허용

2.5 비밀번호 확인 기능 (SignupPasswordConfirmActivity)

2.5.1 정확성 검증
- 비밀번호 입력 오류 방지를 위한 이중 확인 시스템
- 개인 데이터 보안을 위한 정확한 인증 정보 확보

2.5.2 검증 로직
- 이전 단계 비밀번호와 실시간 일치 여부 확인
- 마스킹/언마스킹 기능으로 사용자 편의성 제공
- 시각적 피드백으로 일치 여부 즉시 확인

2.6 개인정보 입력 기능 (SignupProfileActivity)

2.6.1 AI 개인화를 위한 기본 프로필 수집
- **성별**: AI 웹툰 생성 시 캐릭터 설정 및 스토리 톤 결정에 활용
- **생년월일**: 연령대별 콘텐츠 선호도 및 스토리 스타일 개인화에 활용

2.6.2 데이터 활용 목적
- 수집된 정보는 AI 알고리즘의 개인화 파라미터로 사용
- 연령과 성별에 적합한 웹툰 스타일 및 스토리 톤 자동 조정
- 향후 다이어리 분석 시 개인적 특성 반영을 위한 기초 데이터

2.7 장르 선택 기능 (SignupGenreActivity) - **AI 스토리 스타일 결정**

2.7.1 AI 웹툰 생성을 위한 스토리 선호도 수집
- **최대 5개 장르 선택**: AI가 사용자 다이어리를 웹툰으로 변환 시 적용할 스토리 스타일 결정
- **다중 선택 허용**: 복합적 스토리 스타일 지원으로 더 풍부한 웹툰 생성 가능

2.7.2 장르 카테고리별 AI 적용 방향
1. **로맨스**: 감정적, 따뜻한 톤의 웹툰 스타일 / 인간관계 중심 스토리 해석
2. **액션/판타지**: 역동적, 모험적 요소 강조 / 일상을 판타지적으로 재해석
3. **드라마/일상**: 현실적, 감성적 스토리텔링 / 평범한 일상의 의미 부각
4. **스릴러/공포**: 긴장감, 미스터리 요소 가미 / 일상 속 흥미진진한 요소 발굴

2.7.3 AI 개인화 활용
- 선택된 장르 조합에 따른 웹툰 스토리 방향성 결정
- 사용자 다이어리 텍스트 분석 시 선호 장르 필터 적용
- 캐릭터 설정 및 배경 스타일에 장르 특성 반영

2.8 그림체 선택 기능 (SignupArtStyleActivity) - **AI 비주얼 스타일 결정**

2.8.1 AI 웹툰 생성을 위한 비주얼 스타일 선택
- **6가지 그림체 중 1개 선택**: AI가 웹툰 생성 시 적용할 아트 스타일 결정
- **단일 선택**: 일관된 비주얼 아이덴티티 유지를 위한 하나의 대표 스타일 선택

2.8.2 그림체별 AI 생성 특성
- style1~style6: 각각 다른 드로잉 스타일과 색감 적용
- 선택된 그림체는 사용자 전용 AI 모델 파라미터로 설정
- 향후 모든 웹툰 생성 시 일관된 아트 스타일 적용

2.8.3 회원가입 완료 및 AI 프로필 구축
- 수집된 모든 정보(장르 + 그림체 + 개인정보)를 종합하여 개인화 AI 프로필 생성
- 메인 서비스(다이어리 작성 및 웹툰 생성)로 진입
- 사용자 맞춤형 AI 웹툰 생성 준비 완료

4. 비기능 요구사항 (Non-Functional Requirements)
================================================================================

4.1 성능 요구사항
- 회원가입 플로우 완료 시간: 5분 이내
- 페이지 전환 시간: 300ms 이내 애니메이션
- 이메일 검증 응답 시간: 1.5초 이내
- 메모리 사용량: 50MB 이하 (현재 단계 기준)

4.2 사용성 요구사항
- 직관적인 UI/UX (One Page One Action 원칙) - AI 서비스 진입 장벽 최소화
- 실시간 피드백 제공 - 사용자 가이드 및 오류 방지
- 명확한 진행 상태 표시 - 회원가입 과정의 투명성 확보
- AI 서비스 특성을 반영한 친숙한 인터페이스

4.3 확장성 요구사항 (향후 AI 기능 연동 대비)
- 사용자 선호도 데이터 구조 확장 가능성
- AI 모델 연동을 위한 API 인터페이스 준비
- 대용량 웹툰 콘텐츠 처리를 위한 아키텍처 고려

4.4 보안 요구사항 (개인 다이어리 데이터 보호)
- 사용자 개인정보 암호화 저장
- 다이어리 텍스트 데이터 보안 처리 준비
- AI 모델 학습 시 개인정보 비식별화

5. 데이터 요구사항
================================================================================

5.1 사용자 프로필 데이터 (AI 개인화 기초)
- 이메일: String (계정 식별자)
- 비밀번호: String (해시 암호화 저장)
- 성별: String ("남성" | "여성") - AI 캐릭터 설정용
- 생년월일: String (YYYY-MM-DD) - 연령대별 콘텐츠 개인화용
- **선택 장르**: ArrayList<String> (최대 5개) - **AI 스토리 스타일 결정용**
- **선택 그림체**: String (style1~style6) - **AI 비주얼 스타일 결정용**

5.2 AI 개인화 프로필 생성
- 수집된 선호도 데이터를 기반으로 사용자별 AI 생성 파라미터 구성
- 장르 조합에 따른 스토리 가중치 설정
- 그림체 선택에 따른 비주얼 스타일 템플릿 연결

5.3 데이터 활용 계획
- Phase 2: 다이어리 작성 시 개인화 추천 제공
- Phase 3: AI 웹툰 생성 알고리즘 입력값으로 활용
- Phase 4: 사용자 행동 분석 및 서비스 개선에 활용
